# Install only on the Raspberry Pi (pip install -r requirements-modem.txt),
# not on the Mac dev machine - not needed there, since the modem daemon only
# ever runs on the Pi.
#
# Using lgpio rather than RPi.GPIO as the GPIO backend: RPi.GPIO is a legacy
# library that's increasingly awkward on Bookworm-era kernels (the GPIO
# character-device interface changed) and often fails to build against
# newer Python versions. gpiozero auto-detects and uses lgpio with no code
# changes needed on our side.
#
# pyserial is NOT needed: modem/serial_transport.py talks to the AT ports
# directly via os/termios/select. This was a deliberate switch away from
# pyserial - its normal open() sequence touches DTR-related modem-control
# lines that this module's ttyUSB2 interface doesn't support, causing a
# BrokenPipeError there. The custom transport avoids that entirely.
gpiozero>=2.0
lgpio>=0.2.2
